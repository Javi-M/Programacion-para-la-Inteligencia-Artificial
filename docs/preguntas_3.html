La usamos sobre la función de error: el error en función de los parámetros.

Sirve para minimizar el error.

Sea \(\mathcal L\) la función de error (campo escalar: \(\mathcal L: \mathbb R^n \to \mathbb R\)). Es decir hay \(n\) parámetros y el error depende de ellos. Con el gradiente, podemos saber para un punto (unos ciertos parámetros) la derivada direccional máxima: es decir, en qué dirección incrementa más el error dado unos parámetros. (Por tanto, querremos modificar los parámetros en la dirección contraria).

Acarrea el inconveniente de que \(\mathcal L\) tiene que ser diferenciable en todos sus puntos: esto hará que haya que usar ciertas técnicas.

Recordar que se deriva respecto los parámetros, no sobre los valores de entrada.

¿A qué denominamos forward pass (pasada hacia adelante)? ¿Y backwards pass (pasada hacia detrás)? (De forma general).

TipSolución

Forward pass: el modelo hace una predicción.

Backwards pass: cálculo del gradiente del modelo.

¿Mediante qué calcula PyTorch los gradientes?

TipSolución

Mediante Autograd. https://docs.pytorch.org/docs/2.13/autograd.html

¿Para qué sirve la opción requires_grad? ¿Fallará el siguiente código? ¿Por qué?

x = torch.tensor(2.0, requires_grad=False)
x.backward()

¿No sería mejor dejarlo siempre a True?

TipSolución

Indica si el gradiente debe ser calculado para el Tensor. (Para el sistema Autograd de PyTorch).

backward() calcula el gradiente del tensor, y hemos dicho que no queremos que sea calculado, obviamente fallará.

Si siempre se pusiera a True, iría guardando información de los tensores para las derivadas posteriores, lo que aumentará la memoria usada. Solo interesan los gradientes de aquellos tensores que se quieran optimizar o de los que se necesiten derivadas.

¿Qué funciones de PyTorch implementan el error absoluto y el error cuadrático?

TipSolución

torch.nn.L1Loss() y torch.nn.MSELoss().

Al calcular el error entre dos tensores \(\hat y\), \(y\), hace el error elemento a elemento y luego la media. Esa forma de combinar valores es lo que se conoce como reduction.

loss_function = torch.nn.MSELoss(size_average=None, reduce=None, reduction='mean')
